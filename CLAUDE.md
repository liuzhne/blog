## Commands

- `npm run dev` — local dev server at `localhost:4321`
- `npm run build` — static build to `./dist/`
- `npm run preview` — serve the built `./dist/`
- `npm run astro check` — type-check `.astro` files (Astro's strict tsconfig)
- Node `>=22.12.0` is required (matches the GitHub Actions runner).

There is no test runner or linter configured in this project.

## Architecture

Astro 6 static site (Chinese-language personal blog) deployed to GitHub Pages at `https://liuzhne.github.io`.

- **Tailwind v4 via Vite plugin** (`@tailwindcss/vite`), wired in `astro.config.mjs`. Styles live in `src/styles/global.css`, which is `@import`-ed by `src/layouts/Layout.astro` — there is no `tailwind.config.js` and no PostCSS config; v4 reads styles directly from the CSS file.
- **`trailingSlash: 'always'`** is set in `astro.config.mjs`. All internal links must end with `/` (e.g. `/blog/posts/foo/`, not `/blog/posts/foo`), or routing breaks under the static build.
- **Content collection** `posts` is defined in `src/content.config.ts` using the `glob` loader against `src/content/posts/**/*.md`. Schema requires `title`, `description`, `pubDate`; optional `updatedDate`, `tags`, `author`, `draft`. The blog index filters out `draft: true` and sorts by `pubDate` desc.
- **Dynamic post routes** are generated by `src/pages/blog/posts/[...slug].astro` via `getStaticPaths()` over the collection, with `params.slug = post.id`. The post body is rendered through `src/layouts/BlogPost.astro`, which wraps `src/layouts/Layout.astro`.
- **Layout chain**: `Layout.astro` (HTML shell, SEO meta, Open Graph, dark-mode bootstrap) → `Navigation` + `<slot/>` + `Footer`. `BlogPost.astro` adds the article header/footer around `<slot/>`.
- **Dark mode** is class-based (`.dark` on `<html>`). The pre-hydration inline script in `Layout.astro` reads `localStorage.theme` (or system preference) before paint to avoid FOUC; toggle logic should write the same key.

## Deploy

`.github/workflows/deploy.yml` runs on push to `main`: cleans `.astro` and `dist`, runs `npm ci && npm run build`, and uploads `./dist` to GitHub Pages. There is no preview environment — `main` is production.
